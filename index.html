What is Kubernetes?

Often abbreviated as K8s, Kubernetes is an open-source container orchestration engine designed to automate deploying, scaling, and managing containerized applications.

Pods

The smallest deployable units of computing you can create and manage in Kubernetes. A Pod contains one or more containers (like Docker containers) sharing storage and network resources.

Read docs

Nodes

A node is a worker machine (either physical or virtual) in Kubernetes. It contains the services necessary to run Pods (like the kubelet and a container runtime) and is managed by the control plane.

Read docs

Deployments

A declarative configuration file that defines how many replicas of a Pod should run, handles rolling updates, and automatically replaces failed instances to maintain high availability.

Read docs

Services

An abstract way to expose an application running on a set of Pods as a network service. It provides a single stable IP address and DNS name, routing requests across dynamic, ephemeral Pods.

Read docs

Windows 11 Prerequisites Checklist

Ensure your system is configured correctly before starting the installation. Toggle the checklist to track your progress.

Windows 11 (Home, Pro, or Enterprise)
Minikube works on all editions. Build 22000 or higher is recommended.
BIOS Virtualization Enabled (VT-x / AMD-V)
Required to launch hypervisors or containers. Check Performance tab in Task Manager.
Hardware Specifications (2+ CPUs, 4GB+ RAM)
Minikube requires at least 2 CPUs, 4GB of memory, and 20GB of free disk space.
Container Engine or Hypervisor Installed
Install Docker Desktop (Recommended), Hyper-V, or VirtualBox. Docker is the default driver.

Step-by-Step Installation

Follow these steps to download the Kubernetes CLI, install Minikube, and launch your first cluster using PowerShell.

1

Install Kubernetes CLI (kubectl)

The kubectl command line tool allows you to run commands against Kubernetes clusters. You can install it quickly using the Windows package manager (winget) or manually.

PowerShell Command

Run your terminal (PowerShell or Command Prompt) as **Administrator** to perform system-wide winget installs.

PowerShell 
PS C:\>winget install Kubernetes.kubectl

Alternative (Direct Download): Download the latest kubectl.exe and add the folder containing it to your system PATH.

2

Install Minikube

Minikube is local Kubernetes, focusing on making it easy to learn and develop for Kubernetes. It runs a single-node Kubernetes cluster inside a container or VM on your PC.

PowerShell 
PS C:\>winget install Kubernetes.minikube

Note: Restart your terminal after these installations to reload environment variables and ensure commands are registered.

3

Verify CLI Tools

Open a new PowerShell window and execute the verification commands to confirm both binaries are active on the PATH.

PowerShell 
# Verify kubectl version
PS C:\>kubectl version --client

# Verify minikube version
PS C:\>minikube version
4

Launch Minikube Cluster

Now, run minikube by specifying the driver. In Windows 11, the Docker driver is recommended and is set by default if Docker Desktop is running. Alternatively, you can use Hyper-V.

Docker Desktop Configuration

Make sure Docker Desktop is started and configured for Linux containers before running the Docker driver command.

PowerShell 
# Start with Docker driver (Recommended)
PS C:\>minikube start --driver=docker

# Start with Hyper-V driver (Alternative)
PS C:\>minikube start --driver=hyperv

CLI Command Cheat Sheet

Quick reference guide for common commands you will use daily with Minikube and kubectl.

Command Group Action Command Snippet
Minikube Control Start cluster using local engine minikube start
Minikube Control Pause Kubernetes control plane nodes minikube pause
Minikube Control Shutdown the Minikube cluster VM/node minikube stop
Minikube Control Completely delete local Minikube files/node minikube delete
Cluster State Retrieve list of nodes in cluster kubectl get nodes
Cluster State List active application Pods in default namespace kubectl get pods
Deployments Create a new application deployment kubectl create deployment nginx --image=nginx
Deployments Expose application as network service kubectl expose deployment nginx --port=80 --type=NodePort
Diagnostics View details of a specific Pod kubectl describe pod [pod-name]
Diagnostics Print logs for a container inside a Pod kubectl logs [pod-name]
